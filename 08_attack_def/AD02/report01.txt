Кракий аудит по настройкам безопасности AD (методики приказов ФСТЭК 239, 17, 21)

1. Не настроена паролтная политика и политики блокировки учетных записисей с учетом требований безопасности
Политики -> Конфигурация Windows -> Параметры безопасности -> Политики учетных записей:
1.1 Минимальная длинна пароля равна 7, требования от 8, а на серверах от 16
Не включен аудит минимальной длинны пароля
1.2 Пороговое значение блокировки равно 0, требование 5
1.3 Продолжительность блокировки на определена, должно быть 5 минут
1.3 Не установлено разрешение блокировки учетной записи администратора

2. Не настроины локальные политики
Политики -> Конфигурация Windows -> Параметры безопасности -> Локальные политики:
2.1 Политика аудита не задана, по требованиям безопасности должна быть настроина регистрация событий безопасности АУД0, АУД4, УАД9.
2.2 Параметры безопасности не настроены (связанные с интерактивным входом в систему, с контролем учетных записей, сетевой безопасностью, сетевым доступом) - УПД6, УПД7, УПД8, УПД9, УПД10, УПД13, АУД5
2.3 Не задана политика прав пользователя - требования УПД4, УПД7
2.4 Не настроены:
Предупреждение пользователя при его доступе к информационным ресурсам
Ограничение числа параллельных сеансов доступа
Блокирование сеанса доступа пользователя при неактивности

3. Не настроина политика аудита
3.1 Вход учтеной записи (аудит учетных записией) - обязательное требование
3.2 Управление учетными записями (аудит групп) - обязательное требование
3.3 Подробное отслеживание (аудит процессов)
3.4 Доступ к службе каталогов (DC) (аудит доступа к службе каталогов) - обязательное требование
3.5 Вход/выход (аудит блокировки учетных записей, входа, выхода) - обязательное требование
3.6 Доступ к объектам (реестр, съемные носители и т.д) 
3.7 Изменение политики (аудит измениния политики аудита, автоизации) - обязательное требование
3.8 Использование привилегий (аудит использования привилегий) - обязательное требование
3.9 Система (аудит состояния безопасности системы)
3.10 Аудит доступа к глобальным объектам

4. Журналы событий
4.1 Доступ к журналам событий безопасности должен быть ограничен

5. Разделение полномочий (ролей) пользователей
5.1 Администраторы домена должны управлять только доменом и GPO, администраторы серверов имеют доступ к управлению только серверами, локальные администраторы могут управлять только локальными коипьютерами
5.2 Не определены политики упраления доступа (дискреционная, мандатная, ролевая, комбинированная)
5.3 Нет разделения функций по управлению (администрированию) информационной (автоматизированной) системой с иными функциями
