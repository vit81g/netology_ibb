1. Что из того, что предлагает SELinux, может быть реализовано в стандартном Linux?
Вы можете ссылаться на любую политику, расположенную в каталоге /etc/selinux. Могут использоваться три основные политики:
а) targeted - защищает основные системные сервисы, например, веб-сервер, DHCP, DNS, но не трогает все остальные программы.
б) strict - самая строгая политика, управляет не только сетевыми службами, но и программами пользователя.
в) mls - содержит не только правила, но и различные уровни безопасности. Она позволяет реализовать многоуровневую систему безопасности на основе SELinux.

Для стандартного Linux можно реализовать политику targeted, что улучшит безопасность системы, но не сильно скажеться на пользователях. С помощью политики targeted создадим модели минимально необходимых привилегий для каждого сервиса, пользователя и программы.

2. Можно ли установить SELinux поверх существующей ОС Linux?
SELinux это новый метод контроля доступа в Linux на основе модуля ядра Linux Security (LSM). SELinux включен по умолчанию во многих дистрибутивах на основе Red Hat, использующих пакетную базу rpm, например, Fedora, CentOS и т д.
Если SELinux не установлена, то можжно установить с помощью команды: sudo apt install policycoreutils
