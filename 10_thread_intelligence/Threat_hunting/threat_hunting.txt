Цель -
обнаружить сложные угрозы до момента реализации цели атаки. Угрозы
(предположительно) не детектируются СЗИ ИБ

Свойства:
неавтоматический
основан на гипотезах
зависит от опыта

Время пребывания в сети:
Атакующий:
1) первичная разведка, 2) подготовка ВПО, 3) доставка ВПО, 4) первичное исполнение,
5) проникновение в сеть, внутренняя разведка, запуск дополнительного ВПО, доступ
к ключевым ресурсам и т.д.

Защитник:
На пятой стадии атакующего должен применять защитные меры

Dwell time (время пребывания в сети) - это время от начала действий атакующего и
обнаружения СЗИ защитника

Threat Hunting предназначен для поиска угроз которые могут обходить СЗИ и уменьшать
 время пребывания в сети атакующего

Цикл проактивного поиска угроз
1) сбор информации об угрозах
2) создание гипотезы о действиях атакующего (техники, группы)
3) сбор информации для проверки гипотезы
4) фильтрация информации (проверка хостов, процессов, base line)
5) проверка гепотизы (поиск признаков активности в сети, цели запуска процессов (легитимных))


*********Модели и источники информации об угрозах***********
1. Cyber Kill Chain
Состав (Phases of the intrusion Kill Chain):
1) Reconnaissance (подготовка, разветка. Сбор информации сканирование (активное)
OSINT (пассивное))
2) Weaponization (подготовка ВПО, эксплоиты, полезная нагрузка. Подстройка под
жертву (сотрудника), подготовка серверной инфраструктуры для отправки команд ско-
прометированным хостам)
Thread hunting вступает в силу
3) Delivery (доставка. Фишиг, атака на популярный ресурс, атака на цепочку поставок,
эксплуатация уязвимости сервера (публичного))
4) Exploitation (эксплуатация, для повышения привелегий. Эксплойты, доступ к учетным
 данным администратора (самое частое, обнаружение в памяти), использование ошибок
 конфигурации. Все это дает доступ для запуска ПО от имени админа)
5) Installation (Цель - постоянный доступ к системе. Установка backdoor,
предоставление постоянного доступа. Постэксплуатационные фреймворки - Cibalt Strike,
 Metasploit, PowerShell Empire. Бэкдор APT-группы (редкие, недоступные). Прямой
 доступ к зараженной системе - vpn, удаленное управление и т.д (простота).
 Известный бот - Qbot, Trickbot, IcedID, Emolet)
6) Command & Control (коммуникации с управляющим сервером необходимы для управления
загруженной системой и передачей команд. Известные протоколы (HTTP, HTTPS, DNS,
и т.д.). Коммуникации через переносные устройства (флешки - для систем где нет
прямого подключения к сети интернет). Специальные протоколы (известный протокол
+ шифрование).)
7) Action on Objective (атакующие предпринимают действия для достижения финальной
 цели атаки. Кража данных (шпионаж). Уничтожение данных (шифровальщики). Кража
 денег. Продвижение внутрь сети (сканирование, поиск новых целей).)

Thread intellegence покрывает 1 - 2 этапы
Thread hunting  покрывает 3 - 7 этапы
3) Delivery  - Искать подозрительные письма (фишинг)
4) Exploitation - Искать подозрительное ПО (необходимы данные с хоста для анализа,
 что бы признать ПО подозрительным, запуск ПО с подозрительными аргументами)
5) Installation - Искать новые исполняемые файлы (файлы должны быть запущены или
атакующий создает постоянный запуск для бэкдора)
6) Command & Control - Искать подозрительное использование сети (процесс активно
коммуницирует с какими-то странными IP адресами, нужна инфа с хостов, сетевая
информация)
7) Action on Objective - Искать новые архивы (зависит от атакующего (APT))

Но это условно. Модель не является основой. Все меняется, в зависимости от атакующего
