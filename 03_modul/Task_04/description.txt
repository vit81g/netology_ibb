ufw disable МЭ выключен
Установка соединения
1. отправляем SYN пакет с ip 192.168.0.2 на ip 192.169.0.1:21  
2. приходит SYN, ACK пакет с ip 192.168.0.1:21 на ip 192.168.0.2 
3. отправляем ACK пакет (подтверждение) с ip 192.168.0.2 на 192.168.0.1:21
Соединение установлено
4. приходит response (отклик) с ip 192.168.0.1:21 на ip 192.168.0.2 протокол соединения FTP
5. отправляем ACK пакет с ip 192.168.0.2 на ip 192.169.0.1:21
Завершение соединения
6. отправляем FIN, ACK пакет с ip 192.168.0.2 на ip 192.169.0.1:21
7. приходит FIN, ACK пакет с ip 192.168.0.1:21 на ip 192.168.0.2
8. отправляем ACK пакет с ip 192.168.0.2 на ip 192.169.0.1:21 
Соединение закрыто

ufw enable МЭ включен
1. отправляем SYN пакет с ip 192.168.0.2 на ip 192.169.0.1:21
2. TCP retransmission (повторная передача) пакета с ip 192.168.0.2 на ip 192.169.0.1:21
3. TCP retransmission (повторная передача) пакета с ip 192.168.0.2 на ip 192.169.0.1:21

Итог:
Разница в процессе соединения при включенном МЭ и выключенном МЭ проявляется в том, что при выключенном МЭ на запрос об установке соединения инициатор получает ответ в виде пакет SYN, ACK с последующим соединением, а при включенном МЭ инициатор соединения не получает ответ на запрос и отправляет его повторно (TCP retransmission)
