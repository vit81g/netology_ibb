Задача №1. Меры безопасности
1) Сколько уровней безопасности предусмотрено?
Google Cloud предусмотрено шеть уровней безопасности
а) Первый - границы собственности. Это вывески, физические ограждения с указателями
б) Второй - периметр безопасности. Это система физической защиты (видеонаблюдение, охрана, ограждения.
в) Третеий - пропускной режим. Система идентификации личности по уникальному признаку
г) Четвертый - Центр управления безопасностью. Обработка данных видеокамер, входа выхода со считывателей, сканирование уникальных признаков.
д) Пятай - особая зона с доступом только по необхоимости и по категориям (инженеры, техники). Доступ только к оборудованию, не к информации (информация зашифрована).
е) Шестой - уничтожение оборудования с конфиденциальными данными. Доступ в зону имеют только назначенные специалисты.

2) Как вы думаете, зачем и исходя из чего выполнено разбиение на эти уровни?
Для физической защиты - предусмотрены система физ защиты, пропускной режим, идентификация сотрудников, контроль действий. 
Для защиты информации в ЦОД - сотрудники имеют ограниченный доступ к аппаратным средствам ЦОД, все информация в зашифрованном виде, предусмотрена операция полного унчтожения информации на носителе. 
Для надежность операций в ЦОД - сбор серверов только по спецификации Google (отсутствуют в продаже), ОС Linux только с необходимым набором программ, бэкапы, дублирование информации по распределенным ЦОДам, резервные источники питания. 
Исходя из конфиденциальности, целостности, доступности. 
Данное разбиение необходимо для раграничение ответственности, каждый отвечает за свою зону - система физической охраны; система обслуживания, модернизации оборудования; системы распределения(репликации), защиты данных, бэкапов и уничтожение данных.

3) Опишите, какие меры безопасности предпринимаются и предположите для каких сценариев (это должно быть явно видно из принимаемых мер)  
а) Защита от несанкционированного физического проникновения посторонних на территорию, где ведется обработка данных. Сценарий физичческого проникновения с целью кражи. порчи оборудования
б) Защита от физического несанкционированного доступа в зону, где ведется обработка информации. Система контроля доступа. Сценарий, где внутренний нарушитель может получить доступ в помещение, где вдется обработка информации с целью копирования, модификации, уничтожения.
в) Обработка всех инцендентов касательно системы физической защиты. Сценарий когда нарушитель по чужому пропуску или с помощью сотрудника компании может проникнуть в помещение с ограниченным доступом, на территорию.
г) Ограничение доступа в особые зоны, вход только по необходимости или всопровождении. Сценарий - уборщица зашла в серверную и все сломала.
д) Вся информация зашифрована. Сценарий когда техник или инженер имея доступ для выполнения работ может скопировать информацию.
е) Контроль уничтожения информации. Сценарий, когда на списанном оборудовании можно извлечь информацию для дальнейшего использования.
